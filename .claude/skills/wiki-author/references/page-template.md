# Page template

Use this as a starting skeleton for a typical feature page. Drop sections that don't apply.

```markdown
# <Feature name>

> One-sentence description of what this feature is and the goal it serves.

![<short caption>](<raw github url to screenshot>)

## When to use it

A short paragraph or bullet list framing the user's situation. "Use this when…", "This is the right tool when…". Helps readers self-select.

## Open it

The exact path to get to the feature. Menu names, button labels, keyboard shortcuts.

- Menu: **<Menu> → <Item>**
- Shortcut: `Cmd+<X>` (macOS), `Ctrl+<X>` (Windows)
- Toolbar: the <name> icon in the <location>

## How it works

Describe the UI top-to-bottom or left-to-right. Use a screenshot if there is one.

### <Sub-area 1>

What it does, what each control means.

### <Sub-area 2>

Same.

## Common tasks

Small numbered recipes for the things users actually do. One per heading.

### <Task 1>

1. Step one.
2. Step two.
3. Step three.

### <Task 2>

1. …

## Tips & shortcuts

Small bullet list. Keyboard shortcuts, hidden features, gotchas worth knowing.

## Limitations

Honest list of what this feature does not (yet) do. Users appreciate this — it saves them from hunting for a feature that isn't there.

## Related

- [[Other Page]] — when you'd switch to this instead.
- [[Another Page]] — common follow-up workflow.
```
